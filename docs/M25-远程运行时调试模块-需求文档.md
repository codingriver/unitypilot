# M25-远程运行时调试模块-需求文档

> 基于 [RShell](https://github.com/CharlesFeng207/RShell) 协议的远程设备运行时调试桥接。

---

## 1. 模块职责

- 负责：
  - 通过 UDP 协议与运行在真机设备（Android / iOS / Desktop Build）上的 RShell 运行时建立通信连接。
  - 将 MCP 工具调用转换为 RShell 表达式并发送到目标设备，接收并解析响应。
  - 管理连接生命周期：握手（handshake）、心跳检测、断线重连、超时处理。
  - 接收分片响应（Fragment）并在 Python 侧完成重组。
  - 提供对远程设备的场景层级查询、对象属性读写、方法调用能力。
- 不负责：
  - 在远程设备上安装或部署 RShell 运行时（需用户预先集成到 Build 中）。
  - 替代 M19（CSharp代码执行）或 M20（反射调用）在 Editor 内的功能。
  - 远程设备上的文件系统操作（读写本地文件、日志导出等）。
  - 远程设备之间的通信中转。
- 输入：MCP 工具调用参数（设备地址、端口、表达式字符串）。
- 输出：远程执行结果字符串或结构化错误信息。

## 2. 数据模型

### 2.1 连接配置

| 字段名 | 类型 | 语义 | 合法范围 | 默认值 |
| --- | --- | --- | --- | --- |
| connectionId | string | 连接唯一标识 | UUID 格式 | 自动生成 |
| host | string | 目标设备 IP 地址 | 合法 IPv4 地址 | `127.0.0.1` |
| port | int | 目标设备 RShell 监听端口 | [1024, 65535] | 9999 |
| status | string | 连接状态 | disconnected / connecting / connected / error | disconnected |
| lastHeartbeatAt | long | 最近一次心跳成功的 Unix 时间戳 | >= 0 | 0 |
| timeoutMs | int | 单次请求超时时间（毫秒） | [1000, 60000] | 10000 |
| maxRetries | int | 单次请求最大重试次数 | [0, 20] | 3 |

### 2.2 执行请求

| 字段名 | 类型 | 语义 | 合法范围 | 默认值 |
| --- | --- | --- | --- | --- |
| requestId | string | 请求唯一标识 | UUID 格式 | 自动生成 |
| connectionId | string | 目标连接标识 | 已存在的 connectionId | 无 |
| expression | string | RShell 表达式 | 长度 1-10000 字符 | 无 |
| status | string | 请求状态 | pending / sent / completed / failed / timeout | pending |
| result | string | 执行结果序列化文本 | 任意字符串 | 空 |
| error | string | 错误描述 | 任意字符串 | 空 |
| roundTripMs | long | 从发送到收到完整响应的耗时（毫秒） | >= 0 | 0 |

### 2.3 响应分片（内部模型，不对外暴露）

| 字段名 | 类型 | 语义 | 合法范围 | 默认值 |
| --- | --- | --- | --- | --- |
| MsgId | int | 消息标识 | >= 0 | 无 |
| FragIndex | int | 当前分片索引 | [0, FragCount-1] | 无 |
| FragCount | int | 总分片数 | >= 1 | 无 |
| Content | string | 分片内容 | 任意字符串 | 无 |
| CheckMsg | string | 校验消息（完整内容的 MD5 或原文） | 任意字符串 | 无 |

## 4. 交互规格

### 4.1 连接流程

- **触发事件**：客户端发送 `rshell.connect` 消息。
- **状态变化**：
  1. 状态从 `disconnected` 转为 `connecting`。
  2. 向目标 host:port 发送 UDP 握手包，内容为 `hi`。
  3. 收到 `welcome` 响应后状态转为 `connected`。
  4. 超过 timeoutMs 未收到响应，状态转为 `error`，error 字段写入 `"握手超时，未收到 welcome 响应"`。
- **数据提交时机**：状态转为 `connected` 或 `error` 后立即返回结果。
- **取消/回退**：连接过程中不可取消，只能等待超时后重新发起。

### 4.2 表达式执行流程

- **触发事件**：客户端发送 `rshell.execute`。
- **状态变化**：
  1. 创建 requestId，status 设为 `pending`。
  2. 通过 UDP 发送表达式字符串，status 设为 `sent`。
  3. 接收响应分片，逐片缓存。
  4. 所有分片收齐后重组为完整结果，status 设为 `completed`。
  5. 超时未收齐分片，status 设为 `timeout`。
  6. 分片校验失败，执行重试（最多 maxRetries 次），全部失败后 status 设为 `failed`。
- **数据提交时机**：分片重组完成或进入终态后立即返回。
- **取消/回退**：远程表达式一旦发送无法取消执行，仅能停止等待响应。

### 4.3 心跳机制

- 连接建立后每 5000ms 发送一次 `hi` 作为心跳包。
- 连续 3 次心跳无响应（15000ms 无 `welcome` 回复），状态自动转为 `error`，error 字段写入 `"心跳超时，连续 3 次无响应"`。
- 心跳超时后不自动重连，需客户端主动调用 `rshell.connect` 重新建立连接。

### 4.4 高层命令映射

高层 MCP 工具通过以下映射转换为 RShell 表达式：

| MCP 工具 | RShell 表达式 |
| --- | --- |
| `rshell.scene_list` | `Scene.List()` |
| `rshell.scene_info(path)` | `Scene.Info("{path}")` |
| `rshell.get_value(expr)` | `{expr}`（直接发送表达式） |
| `rshell.set_value(expr, value)` | `{expr}={value}` |
| `rshell.call_method(expr, args)` | `{expr}({args})` |

## 6. 校验规则

| 规则 | 检查时机 | 级别 | 提示文案 |
| --- | --- | --- | --- |
| host 格式校验 | rshell.connect 调用前 | Error | `host "{host}" 不是合法的 IPv4 地址` |
| port 范围校验 | rshell.connect 调用前 | Error | `port {port} 超出合法范围 [1024, 65535]` |
| 连接存在性校验 | 所有需要 connectionId 的操作前 | Error | `connectionId "{connectionId}" 不存在` |
| 连接状态校验 | rshell.execute/scene_list/scene_info/get_value/set_value/call_method 调用前 | Error | `连接 "{connectionId}" 当前状态为 {status}，需为 connected` |
| 表达式非空校验 | rshell.execute 调用前 | Error | `expression 不能为空` |
| 表达式长度校验 | rshell.execute 调用前 | Error | `expression 长度 {length} 超出上限 10000 字符` |
| 分片校验失败 | 响应重组时 | Warning | `分片校验失败，正在重试（第 {retry}/{maxRetries} 次）` |
| 响应超时 | 等待响应期间 | Warning | `请求 {requestId} 响应超时（{timeoutMs}ms）` |

## 8. 技术实现要点

### 8.1 关键类与职责

| 类名 | 职责 |
| --- | --- |
| `RShellConnection` | 管理单个 UDP 连接的生命周期：握手、心跳、收发、断开。持有 `socket` 实例。 |
| `RShellConnectionManager` | 管理多个 `RShellConnection` 实例，按 connectionId 索引。 |
| `FragmentAssembler` | 缓存接收到的分片，按 MsgId 分组，收齐后重组为完整字符串。分片超时清理周期为 30000ms。 |
| `ExpressionBuilder` | 将高层 MCP 工具参数（scene_list / get_value / set_value / call_method）转换为 RShell 表达式字符串。 |
| `RShellToolFacade` | Python 侧 MCP 工具入口，调用 `RShellConnectionManager` 方法。 |

### 8.2 核心流程

```
MCP Tool 调用
  -> RShellToolFacade.execute(connectionId, expression)
    -> RShellConnectionManager.get(connectionId)
      -> RShellConnection.send(expression)
        -> UDP socket.sendto(bytes, (host, port))
      -> RShellConnection.receive(timeoutMs, maxRetries)
        -> FragmentAssembler.feed(datagram)
        -> FragmentAssembler.try_complete(msgId)
          -> 成功: 返回完整字符串
          -> 超时: 重试或返回 timeout 错误
    -> 结果返回给 MCP 客户端
```

### 8.3 实现层级

本模块全部在 **Python 侧** 实现（`src/unitypilot_mcp/` 目录下）。不需要 Unity C# 侧新增代码。

原因：
1. RShell 运行时已由用户集成到 Build 中，Unity Editor 不参与通信。
2. UDP 收发在 Python 中使用标准库 `socket` 即可完成。
3. 避免在 Editor 侧引入对远程设备的网络依赖。

### 8.4 性能约束

- 单个 `RShellConnection` 的 UDP socket 使用非阻塞模式（`setblocking(False)`）配合 `select`/`poll` 实现超时。
- 分片缓存内存上限：单个 MsgId 的分片总大小不得超过 10MB。超过时丢弃该消息并返回错误。
- 心跳定时器使用独立线程或 asyncio task，不阻塞主事件循环。
- 并发连接上限：最多同时维护 10 个 `RShellConnection`。超出时拒绝新连接并返回错误 `"并发连接数已达上限 10"`。

### 8.5 禁止项

- 禁止在 Python 侧实现表达式求值逻辑（所有求值由远程 RShell 运行时完成）。
- 禁止将 UDP socket 绑定到 `0.0.0.0`（仅使用 `sendto` / `recvfrom`，不做服务端监听）。
- 禁止缓存远程执行结果（每次调用必须实时发送到远程设备）。

## 9. 验收标准

1. [前置条件] 本机运行含 RShell 的 Unity Build，监听 127.0.0.1:9999 -> [操作] 调用 `rshell.connect(host="127.0.0.1", port=9999)` -> [期望结果] 返回 connectionId，status 为 `connected`。
2. [前置条件] 已建立连接 -> [操作] 调用 `rshell.execute(connectionId, expression="1+1")` -> [期望结果] 返回 result 为 `"2"`，status 为 `completed`。
3. [前置条件] 已建立连接 -> [操作] 调用 `rshell.scene_list(connectionId)` -> [期望结果] 返回当前场景中的 GameObject 层级列表字符串。
4. [前置条件] 已建立连接，场景中存在名为 "Player" 的物体 -> [操作] 调用 `rshell.scene_info(connectionId, path="Player")` -> [期望结果] 返回 Player 对象的组件列表与属性摘要。
5. [前置条件] 已建立连接 -> [操作] 调用 `rshell.get_value(connectionId, expression="UnityEngine.Application.platform")` -> [期望结果] 返回当前平台枚举值（如 `"Android"`）。
6. [前置条件] 已建立连接 -> [操作] 调用 `rshell.set_value(connectionId, expression="UnityEngine.Time.timeScale", value="0.5")` -> [期望结果] 远程设备 timeScale 变为 0.5，返回成功。
7. [前置条件] 已建立连接 -> [操作] 调用 `rshell.call_method(connectionId, expression="UnityEngine.Screen.SetResolution", args="1280,720,false")` -> [期望结果] 远程设备分辨率变更，返回成功。
8. [前置条件] 无 RShell 运行时在监听 -> [操作] 调用 `rshell.connect(host="127.0.0.1", port=9999)` -> [期望结果] timeoutMs 后返回 status 为 `error`，error 为 `"握手超时，未收到 welcome 响应"`。
9. [前置条件] 已建立连接 -> [操作] 远程设备关闭 -> [期望结果] 15000ms 内心跳超时，status 自动转为 `error`。
10. [前置条件] connectionId 不存在 -> [操作] 调用 `rshell.execute(connectionId="invalid", ...)` -> [期望结果] 返回 Error，提示 `connectionId "invalid" 不存在`。

## 10. 边界规范

- **空数据**：
  - expression 为空字符串时，校验规则拦截并返回 Error `"expression 不能为空"`。
  - connectionId 为空字符串时，校验规则拦截并返回 Error `"connectionId "" 不存在"`。
- **单元素**：
  - 单字符表达式（如 `"1"`）能正常发送并返回结果。
  - 仅建立 1 个连接时，所有操作正常执行。
- **上下限临界值**：
  - expression 长度恰好为 10000 字符时，正常发送执行。
  - expression 长度为 10001 字符时，校验拦截并返回 Error。
  - port 为 1024 时正常连接；port 为 1023 时校验拦截。
  - port 为 65535 时正常连接；port 为 65536 时校验拦截。
  - 同时维护 10 个连接时正常工作；第 11 个连接请求被拒绝。
- **异常数据恢复**：
  - UDP 分片丢失导致重组失败时，重试 maxRetries 次后返回 `failed` 状态，清理该 MsgId 的分片缓存。
  - 远程设备返回畸形 JSON（非标准分片格式）时，丢弃该数据报，记录 Warning 日志，不影响后续收发。
  - 连接处于 `error` 状态时，所有执行操作立即返回 Error 而非挂起等待。

## 11. 周边可选功能

- **P1**：支持通过 ADB 端口转发（`adb forward tcp:9999 tcp:9999`）连接 Android 设备。需预留 `transportType` 字段（值：`direct` / `adb_forward`），当 `transportType=adb_forward` 时自动在连接前执行 `adb forward` 命令。
- **P1**：支持连接列表持久化。将活跃连接配置（host/port/transportType）存储到 `~/.unitypilot/rshell_connections.json`，下次启动自动恢复连接。需预留配置文件路径和格式。
- **P2**：支持远程日志流订阅。通过 RShell 注册 `Application.logMessageReceived` 回调，将远程设备日志实时转发到 MCP 客户端。需预留 `rshell.subscribe_logs` / `rshell.unsubscribe_logs` 工具接口。
- **P2**：支持远程性能数据采集。通过 RShell 表达式定期查询 `UnityEngine.Profiling.Profiler` 数据（FPS、内存、DrawCall），以结构化格式返回。需预留 `rshell.perf_snapshot` 工具接口。
